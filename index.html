CSS = Cascading Style Sheet

  1. Lier la page HTML à la feuille CSS

    ajouter une balise <link> dans le <head>

    <link rel="stylesheet" href="styles.css">
  1. Mettre en place le CSS

    Syntaxe générale

Le CSS se mets en place de la manière suivante :

sélecteur {
	propriété1: valeur1;
	propriété2: valeur2;
}

Le sélecteur est l’élément (= la balise) HTML sur lequel la déclaration va être appliquée

La déclaration est l’ensemble des propriétés appliquées au sélecteur. Les propriétés sont délimitées par un “;”

Une propriété est l’élément sur lequel vous appliquer un style

La valeur est la manière dont vous allez styliser.

Par exemple, pour modifier la couleur du texte de tous les paragraphes, la syntaxe CSS à respecter est la suivante :

p {
	color: #ffffff;
}

p représente la balise paragraphe

color représente la couleur de la police

#ffffff est le code de la couleur blanche

Les codes couleur

En HTML, CSS et dans le monde de l’informatique en général, les couleurs sont codées. Cela permet d’obtenir précisément une couleur, tout en restant simple, compacte et surtout standard. Cela est possible en décomposant la lumière en ses 3 couleur primaires : le rouge, le vert et le bleu. Il s’agit des couleurs primaires en physique. En impression couleur, les couleurs primaires sont le jaune, le cyan et le magenta et en peinture, ce sont le rouge, le bleu et le jaune. Pour bien comprendre le système de codage des couleurs, il faut d’abord comprendre le comptage hexadécimal. Et pour mieux comprendre le comptage en hexadécimal, il est plus facile de commencer avec le comptage en binaire.

Les couleur primaires sont donc chacune divisées en 256 teintes différentes, de la moins intense (0) à la plus intense (255). La notation RGB signifie Red Green Blue (RVB pour Rouge Vert Bleu en français). Le code couleur correspond à une série de chiffres indiquant la quantité de rouge, de vert puis de bleu, le tout précédé du symbole #. En décimal, cela donne donc cette structure : #124056244. Pas très pratique à utiliser…

En réalité, l’utilisation de l’hexadécimal permet de réduire le nombre de caractères à utiliser. Ainsi, pour obtenir l’ensemble des chiffres de 0 à 255 (256 nombres différents) en hexadécimal, il ne faut que 2 caractères (contre 3 en décimal). En effet, 25510_{10}10​﻿ se note ff16_{16}16​﻿. Le code couleur se transforme donc en #7c38f4.

Les unités du CSS

En CSS, il existe plusieurs unités pour spécifier les tailles et les distances. Certaines d’entre elles sont plus utilisées que d’autres, pour des raisons d’ergonomie et d’adaptabilité en fonction du support. Ces notions sont essentielles à l’heure actuelles, pour que les sites web soient utilisables aussi bien avec des ordinateurs, des tablettes ou des mobiles.

Voici les différentes unités à connaitre pour ce cours :

UnitéSignification
px (= Pixel)Une unité de mesure fixe qui correspond à un pixel sur l'écran. Les pixels sont des unités absolues, ce qui signifie qu'ils ne changent pas de taille en fonction de la taille de l'écran ou du périphérique.
% (= Pourcentage)Cette unité est relative à la taille du conteneur parent. Par exemple, si vous définissez la largeur d'un élément à 50%, cela signifie qu'il occupera la moitié de la largeur de son conteneur parent.
emUne unité relative à la taille de la police de l'élément. Par exemple, si la taille de police d'un élément est de 16px, 1em équivaudra à 16px, 2em équivaudra à 32px, et ainsi de suite.
remUne unité relative à la taille de la police de l'élément racine (<html>). Cela signifie que 1rem équivaut à la taille de la police de l'élément racine, tandis que 2rem équivaut à deux fois la taille de la police de l'élément racine.
vw (= Viewport Width)Une unité relative à la largeur de la fenêtre d'affichage. 1vw équivaut à 1% de la largeur de la fenêtre d'affichage.
vh (= Viewport Height)Une unité relative à la hauteur de la fenêtre d'affichage. 1vh équivaut à 1% de la hauteur de la fenêtre d'affichage.
vmin (= Viewport Minimum)Une unité relative au plus petit des deux entre vw et vh.
vmax (= Viewport Maximum)Une unité relative au plus grand des deux entre vw et vh.
cm (= Centimètre)Une unité de mesure absolue correspondant à un centimètre.
mm (= Millimètre)Une unité de mesure absolue correspondant à un millimètre.

En HTML et CSS, un "conteneur parent" fait référence à un élément qui englobe d'autres éléments à l'intérieur de la structure HTML. Ce conteneur parent peut être n'importe quel élément HTML, tel qu'une <div>, un <section>, un <article>, etc. L'idée principale est que cet élément est le parent de ses enfants directs.

<div class="container">
    <h1>Titre</h1>
    <p>Paragraphe 1</p>
    <p>Paragraphe 2</p>
</div>

La <div class="container"> est le conteneur parent des éléments <h1>, <p> et <p> qu'elle englobe. En CSS, vous pouvez cibler ce conteneur parent et appliquer des styles à tous ses enfants à l'aide de sélecteurs CSS appropriés.

Les conteneurs parents sont souvent utilisés pour regrouper des éléments connexes, créer des mises en page, définir des sections de contenu et appliquer des styles de mise en forme. Ils sont essentiels pour structurer efficacement une page web et maintenir un code HTML et CSS organisé et maintenable.

En HTML, l'élément racine fait référence à l'élément le plus haut dans la structure de la page web. Cet élément est généralement <html> et représente le conteneur de tout le contenu HTML de la page. Voici à quoi ressemble généralement la structure d'une page HTML de base :

<!DOCTYPE html>
<html lang="fr">
	<head>
	    <meta charset="UTF-8">
	    <meta name="viewport" content="width=device-width, initial-scale=1.0">
	    <title>Titre de la page</title>
	    <!-- Les balises <meta> ci-dessus définissent la configuration de la page -->
	    <!-- Les liens vers les feuilles de style CSS et les scripts JavaScript sont également souvent inclus ici -->
	</head>
	<body>
	    <!-- Tout le contenu de la page HTML est placé ici -->
	    <!-- Cela peut inclure des éléments tels que des balises <header>, <main>, <footer>, etc. -->
	</body>
</html>

Dans cet exemple, <html> est l'élément racine. Il englobe tout le contenu de la page, y compris l'en-tête (<head>) et le corps (<body>). Les éléments <head> et <body> sont des enfants directs de l'élément racine <html>. L'élément racine peut contenir des métadonnées sur la page, des liens vers des feuilles de style CSS, des scripts JavaScript, des informations sur la langue (lang), etc.

Les conteneurs en HTML

Les conteneurs en HTML sont des éléments qui servent à structurer le contenu d'une page web. Ils peuvent encapsuler d'autres éléments HTML pour appliquer un style, une mise en page ou des comportements spécifiques.

BaliseFonction
<div>C'est le conteneur le plus générique. Il est utilisé pour grouper des éléments afin de les formater avec du CSS ou de manipuler leur comportement avec JavaScript. Le <div> n'a pas de signification sémantique en soi.
<section>Représente une section générique d'un document ou d'une application, généralement avec un thème ou une fonctionnalité cohérente, souvent accompagnée d'un titre.
<header>Sert à définir l'en-tête d'une section ou d'une page. Il peut contenir des éléments introductifs ou de navigation, comme des logos, des titres, des menus de navigation.
<footer>Utilisé pour le pied de page d'une section ou d'une page. Il contient souvent des informations comme les droits d'auteur, des liens vers des mentions légales, des contacts, etc.
<nav>Spécifiquement destiné à la navigation. Il englobe un ensemble de liens de navigation, tels que des menus, des tables des matières, des index.
<article>Destiné à encapsuler un contenu autonome qui pourrait être distribué indépendamment du reste du site, comme des articles de blog, des posts de forum, des commentaires.
<aside>Utilisé pour marquer des contenus tangentiels au contenu principal, comme des barres latérales, des encadrés, des publicités.
<main>Spécifie le contenu principal d'un document. Il ne doit y avoir qu'un seul élément <main> par document, qui contient le contenu unique et central de la page.

Chacun de ces conteneurs a sa propre signification sémantique et doit être utilisé selon le type de contenu qu'il est censé représenter, ce qui aide à améliorer l'accessibilité du web et le référencement naturel (SEO).

ID et classes

Les ID et les classes sont des sélecteurs utilisés en HTML et CSS pour identifier et appliquer des styles spécifiques à des éléments de la page web. Bien qu'ils servent tous deux à personnaliser l'apparence et le comportement des éléments HTML, ils ont des caractéristiques et des utilisations distinctes.

ID

  • Un ID est un identifiant unique qui ne peut être utilisé qu'avec un seul élément HTML sur une page. Cela signifie qu'aucun autre élément sur cette page ne peut avoir le même ID.
  • L'ID est précédé du symbole # lorsqu'il est référencé dans un fichier CSS ou JavaScript.
  • Utiliser un ID quand vous avez besoin de cibler un élément spécifique qui n'apparaît qu'une fois dans le document.

Exemple d'ID :

HTML :

<div id="header">En-tête du site</div>

CSS :

#header {
    background-color: #f0f0f0;
    color: #333;
    padding: 20px;
    text-align: center;
}

Dans cet exemple, l'ID header est utilisé pour appliquer un style spécifique à l'élément div qui représente l'en-tête du site web.

Classes

  • Une classe peut être utilisée sur plusieurs éléments HTML sur une même page, ce qui la rend utile pour appliquer un style cohérent à différents éléments.
  • La classe est précédée du symbole . lorsqu'elle est référencée dans un fichier CSS.
  • Utiliser une classe quand vous voulez que plusieurs éléments partagent le même style.

Exemple de classe :

HTML :

<div class="box">Contenu 1</div>
<div class="box">Contenu 2</div>

CSS :

.box {
    border: 1px solid #ddd;
    padding: 10px;
    margin: 5px;
}

Dans cet exemple, la classe box est utilisée pour appliquer un style de bordure, de marge et de padding identique à deux éléments div différents.

Différence principale

  • Unicité : Un ID est unique à chaque page et ne peut être assigné qu'à un seul élément. Une classe peut être assignée à plusieurs éléments sur la même page.
  • Usage : Les ID sont utiles pour cibler des éléments spécifiques pour les scripts et le style, tandis que les classes sont idéales pour appliquer un style cohérent à plusieurs éléments.

Les ID et les classes sont des outils puissants pour la conception web, permettant de créer des styles précis et de manipuler des éléments avec JavaScript. Utilisés judicieusement, ils facilitent la création de sites web bien structurés et visuellement attrayants.
